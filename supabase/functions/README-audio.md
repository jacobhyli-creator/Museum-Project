# Audio Narration — Deployment Guide

This document covers the server-side pieces of the audio narration feature. The
assistant **cannot** deploy functions, create Storage buckets from the dashboard,
or hold provider API keys — you perform the steps below.

## Overview

- **Premium audio** is generated by a Supabase **Edge Function** (`generate-audio`)
  using a TTS provider (ElevenLabs by default). It uploads an `.mp3` to Supabase
  Storage and records an `audio_narrations` row.
- Audio is **draft** until an admin **approves** it and **sets it active**. Only
  approved + active + ready audio is visible to visitors (enforced by RLS).
- **Non-English** audio requires an **approved translation** of the explanation
  text first. `draft-translation` produces DRAFT translations for admin review;
  nothing is auto-published.
- **Outdated detection**: each narration stores `source_text_hash`. If the
  explanation text later changes, the admin view flags it and the public player
  refuses to play the stale audio.

## 1. Apply the migration

Run `supabase/migrations/0012_audio_narration.sql` in the Supabase SQL Editor
(or via `supabase db push`). It is additive and does not modify existing
explanation text. It:

- adds `language_code` + `translation_status` to `artwork_explanations`
  (existing rows become `language_code='en'`),
- creates `audio_voices`, `audio_narrations`, the `public_audio_voices` view,
  and the `audio_narration_status` view,
- sets RLS (public sees only approved+active+ready audio; admins see all),
- creates the `audio` Storage bucket + policies,
- seeds 2 placeholder voices per language (en/zh/fr/es).

## 2. Set real voice ids

The seeded voices have `provider_voice_id = 'REPLACE_ME_*'`. In the admin **Audio
→ Voices** panel (or SQL), set each voice's real ElevenLabs voice id and, if
desired, `model_name`. Generation refuses to run on a `REPLACE_ME` voice.

## 3. Deploy the Edge Functions

```bash
supabase functions deploy generate-audio
supabase functions deploy draft-translation   # optional (translations)
```

## 4. Set function secrets

```bash
# Required for generate-audio
supabase secrets set TTS_PROVIDER=elevenlabs
supabase secrets set ELEVENLABS_API_KEY=sk_...

# Required for draft-translation (optional feature)
supabase secrets set TRANSLATION_API_URL=https://api.openai.com/v1/chat/completions
supabase secrets set TRANSLATION_API_KEY=sk_...
supabase secrets set TRANSLATION_MODEL=gpt-4o-mini
```

`SUPABASE_URL`, `SUPABASE_ANON_KEY`, and `SUPABASE_SERVICE_ROLE_KEY` are provided
to Edge Functions automatically by the platform.

## 5. Generate → review → approve

1. In **Admin → Audio**, pick an artwork/style/language/voice and click
   **Generate**. The function synthesizes audio, uploads it, and marks the row
   `ready` (still `draft`).
2. **Preview** the audio in the admin panel.
3. **Approve** it and **Set active**. It becomes visible to visitors on their
   next session (the app fingerprints content changes, so no redeploy needed).

## 6. Translations (non-English)

1. **Admin → Audio → Translations**: generate a **draft** translation (or paste
   your own), review it, and **approve** it.
2. Only then can non-English audio be generated for that artwork/style/language.

## Provider-agnostic

To use a different TTS provider, add an adapter in
`generate-audio/providers/<name>.ts` implementing `TtsProvider`, register it in
`providers/index.ts`, and set `TTS_PROVIDER=<name>`. Nothing else changes.

## Security notes

- The browser holds **only** the anon key; it never sees provider keys or the
  service role.
- Both functions verify the caller is an admin/editor (`can_edit_content()`)
  before doing anything.
- RLS guarantees anonymous visitors can read only approved, active, ready audio.
